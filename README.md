> 本仓库用于快速验证机器人管理后台的交互流程与主要页面布局，所有数据为前端模拟，便于 UI/交互评审。

  - 操作记录（已完成）
  - 群组管理（已完成基础）

- 商户管理
  - 商户列表（已完成：新增"钱包地址"字段与筛选）

## 订单管理
- 商户订单管理（已完成：筛选区布局优化）
  - 筛选布局：采用流式 flex 排布（自动换行），行间距 `8px`、列间距 `12px`
  - 控件宽度：下拉统一 `168px`，整体排版更紧凑，避免筛选项独占一行
- 商户提现审核（已完成：新增提现审核页面）
  - 筛选功能：商户名称（输入框，模糊搜索）、提现单号、状态（未确认/成功/失败）、申请时间
  - 表格字段：商户名称、提现订单号、提现数量（U）、类型、钱包地址、提现金额（CNY）、备注、审核备注、状态、创建时间、完成时间
  - 操作功能：拒绝、导出（导出按钮位于工具栏右侧）
  - 操作规则：仅“未确认”状态可操作；拒绝在审核弹窗填写备注；操作列仅保留【拒绝】按钮；暂不提供批量操作
  - 备注展示：根据行数据动态生成固定模板文案，含商户名称、数量*汇率、RMB金额与地址
  - 拒绝弹窗：显示提现详情，仅支持拒绝，输入审核备注
  - 展示修复：类型列统一显示 `TRX`（当前仅支持 TRX）
  - 订单号格式：`MTX` + `yyyyMMddHHmmss` + 6位随机数字（示例：`MTX20251112153045123456`）

  - 表单策略：新增与编辑弹窗中不包含 `TGID` 字段
- 界面优化：移除“普通账号”列表中的“状态”列和操作列中的“编辑”按钮。
- 表单优化：新增与编辑弹窗中移除“状态”字段，新增时默认为启用状态。

## 数据清理
- 页面职责：定期清理系统中的历史数据、日志等，以保持系统性能。支持按时间范围或保留天数清理商户订单、结算单和操作日志，并可按状态进行筛选。
- 按钮显示：当自动清理关闭时显示“执行一次性清理”且不显示“保存设置”；当自动清理开启时不显示“执行一次性清理”且仅显示“保存设置”。
- 交互已落地：已按“自动清理”开关实现按钮显隐逻辑（页面已更新）
 - 按钮样式：两个按钮统一使用`primary`蓝色，视觉一致
 - 默认范围：其他清理模块页面初始为全选（平台分润统计、商户收款统计、商户余额快照、商户资金流水、商户预付记录、供应商资金流水、供应商预付记录、机器人操作记录）
 - 禁用规则：当其他清理范围为空时，“执行一次性清理”按钮置灰不可点击，并提示“请至少选择一个清理范围”。
