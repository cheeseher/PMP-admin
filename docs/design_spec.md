# 设计规范

## 1. 概览
暂无详细规范。请在此维护 UI/UX 规范，防止样式和交互逻辑偏差。

## 2. 规范列表
- 组件层级
- 布局规则
- 配色/字体/间距 Token
- 交互动画逻辑
- 供应商通道 (`Channel.vue`) 表单新增“支持设备”的多选框控件。顶部搜索栏支持单选筛选，表格使用 `el-tag` 不同的类型（info/success/warning）做视觉区分展示所见设备。
- 商户收款统计 (`MerchantIncome.vue`) 页面数据表格列名及数量更新，新增4个业务相关金额列（隔日修改订单金额、手续费、冲账金额及调账后入账金额），这4个列均使用 `amount-cell` 样式以保持等宽数字排版与金额的精确对齐显示。
- **支付产品管理 (`Product.vue`)**：编辑弹窗中的“商户费率”同步功能已优化。同步勾选框从输入框旁移至底部的“同步配置到商户”下方。仅当选择“立即同步”或“定时同步”时，才会展示“同步商户费率”勾选框。**默认不勾选**，由操作员手动确认是否同步费率。

- **拉人弹窗普通账号下拉格式**（GroupManagement.vue、GroupCategoryManagement.vue）：一键拉人和批量拉人弹窗中，普通账号下拉项统一显示为 账号名称（TGID）格式（TGID灰色小字显示于名称后方）；:label 属性绑定为 名称(TGID) 用于折叠标签和搜索匹配。**新增“全选”哨兵项**，支持一键切换全选/取消全选状态。
- **批量拉人弹窗群组展示**：考虑到群组数量可能达到上千个，弹窗中**不再展示具体的群组 Tag 列表**，仅以摘要形式显示“已选 X 个群组”，以保持界面简洁。
- **页面按钮布局**：列表上方的操作按钮（如“新增”、“批量拉人”、“刷新”）应放置在**表格卡片的内部上方工具栏**，而非顶部的筛选条件卡片中。
- **分组删除确认弹窗**（GroupCategoryManagement.vue）：删除确认提示文本统一为：确认删除分组【分组名称】？删除后已关联该分组的群组将变为未分配状态！，使操作员明确感知删除后果。
